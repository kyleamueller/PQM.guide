## Remarks

`Number.RandomBetween` returns a pseudo-random floating-point number uniformly distributed between `bottom` and `top`, both endpoints inclusive. It is equivalent to `bottom + Number.Random() * (top - bottom)`.

Like `Number.Random`, the value regenerates on each refresh. When used inside `Table.AddColumn`, the engine typically evaluates the expression once per row, producing different values per row. However, caching behavior may vary — if consistent per-row randomness is critical, test in your specific environment.

`bottom` must not exceed `top`. Both parameters accept decimal values.

## Examples

### Example 1: Random integer-range value between 1 and 100

```powerquery
Number.RandomBetween(1, 100)
```


### Example 3: Add a random discount column

Input: `Sales`

```powerquery
Table.AddColumn(
    Table.SelectColumns(Table.FirstN(Sales, 4), {"OrderID", "UnitPrice"}),
    "Discount", each Number.Round(Number.RandomBetween(0, 0.3), 2), type number
)
```
