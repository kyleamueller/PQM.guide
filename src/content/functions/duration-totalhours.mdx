## Remarks

`Duration.TotalHours` returns the entire duration as a decimal number of hours. For example, 1 day and 6 hours returns `30.0`. This differs from `Duration.Hours`, which returns only the hours component (e.g., `6` for the same input). Use `Duration.TotalHours` when you need elapsed time as a single numeric value, such as for rate or SLA calculations. If the input is `null`, the function returns `null`.

## Examples

### Example 1: Calculate total hours per order processing time

Compute the total hours elapsed between order and ship date.

Input: `Sales`

```powerquery
Table.AddColumn(
    Table.SelectColumns(
        Table.SelectRows(Sales, each [ShipDate] <> null),
        {"OrderID", "OrderDate", "ShipDate"}
    ),
    "ProcessingHours",
    each Duration.TotalHours(
        DateTime.From([ShipDate]) - DateTime.From([OrderDate])
    ),
    type number
)
```

### Example 2: Total hours for a mixed duration

Return the total hours for a duration spanning 1 day and 6 hours.

```powerquery
#table(
    type table [Duration = duration, TotalHours = number],
    {{#duration(1, 6, 0, 0), Duration.TotalHours(#duration(1, 6, 0, 0))}}
)
```
