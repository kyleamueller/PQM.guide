## Remarks

`Table.StopFolding` breaks the query folding chain. Any transformations applied after this function are evaluated locally in the Power Query engine rather than being translated and sent to the data source (e.g., as SQL). The table's data and structure are unchanged.

Use this function when:
- You want to ensure predictable local evaluation of a step.
- A downstream transformation is incompatible with folding and you want to control exactly where the data is fetched.
- You are debugging and want to isolate which steps fold vs. which are evaluated locally.

Note that breaking folding can significantly increase query execution time for large data sources because all data must be retrieved before local operations begin.

## Examples

### Example 1: Stop folding before a custom transformation

```powerquery
let
    Source  = Sql.Database("server", "db"),
    Orders  = Source{[Schema="dbo", Item="Orders"]}[Data],
    Stopped = Table.StopFolding(Orders),
    Filtered = Table.SelectRows(Stopped, each [Amount] > 100)
in
    Filtered
```

### Example 2: Stop folding to ensure local evaluation

```powerquery
let
    Source  = OData.Feed("https://services.odata.org/V4/Northwind/"),
    Products = Source{[Name="Products"]}[Data],
    Local    = Table.StopFolding(Products)
in
    Table.RowCount(Local)
```
