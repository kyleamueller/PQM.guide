## Remarks

`Text.NewGuid` generates a new version-4 (random) GUID each time it is called and returns it as a lowercase text string in the standard 8-4-4-4-12 hexadecimal format.

Because `Text.NewGuid` is a volatile function (its result changes on each evaluation), Power Query may call it multiple times during a single refresh. If a stable, unique identifier per row is needed, consider generating GUIDs outside Power Query and loading them as data, or using a deterministic surrogate key approach.

In Power Query Online and dataflows, volatile functions may be evaluated differently to support query folding and caching.

## Examples

### Example 1: Generate a single GUID

```powerquery
Text.NewGuid()
```

### Example 2: Add a GUID column to a table

```powerquery
Table.AddColumn(
    #table({"Name"}, {{"Alice"}, {"Bob"}}),
    "ID",
    each Text.NewGuid()
)
```

<!--output
{"columns":[{"name":"Name","type":"text"},{"name":"ID","type":"text"}],"rows":[{"Name":"Alice","ID":"c1a2b3d4-e5f6-7890-abcd-ef1234567890"},{"Name":"Bob","ID":"d4e5f6a7-b8c9-0123-4567-89abcdef0123"}]}
-->

### Example 3: Validate GUID format with Guid.From

```powerquery
Guid.From(Text.NewGuid())
```

<!--output
