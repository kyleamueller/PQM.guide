## Remarks

`Type.IsNullable` returns `true` when a type is marked as nullable — meaning values of that type can be `null`. In Power Query M, nullable types are expressed using the `nullable` keyword, for example `nullable number` or `nullable text`.

The built-in primitive types (like `type number`, `type text`) are non-nullable by default. Their nullable forms (`nullable number`) are distinct type values.

Use `Type.IsNullable` when inspecting type metadata or building generic functions that must handle nullable variants of a type.

## Examples

### Example 1: Check a nullable type

```powerquery
Type.IsNullable(type nullable number)
```
