## Remarks

`Date.IsLeapYear` returns `true` if the year component of the input date is a leap year (i.e., divisible by 4, except centuries unless also divisible by 400), and `false` otherwise. This is equivalent to checking whether February has 29 days in that year. If the input is `null`, the function returns `null`.

## Examples

### Example 1: Flag leap-year orders in the Sales table

Add a boolean column indicating whether the order was placed in a leap year.

Input: `Sales`

```powerquery
Table.AddColumn(
    Table.SelectColumns(
        Table.FirstN(Sales, 5),
        {"OrderID", "OrderDate"}
    ),
    "IsLeapYear", each Date.IsLeapYear([OrderDate]), type logical
)
```

### Example 2: Check specific years

Test 2024 (leap) and 2023 (common).

```powerquery
#table(
    type table [Year = number, IsLeap = logical],
    {
        {2024, Date.IsLeapYear(#date(2024, 1, 1))},
        {2023, Date.IsLeapYear(#date(2023, 1, 1))},
        {2100, Date.IsLeapYear(#date(2100, 1, 1))}
    }
)
```
