## Remarks

`Record.FieldOrDefault` is the safe alternative to `Record.Field` and the `[FieldName]` syntax. When a field does not exist in the record, accessing it with `record[FieldName]` raises an error; `Record.FieldOrDefault` instead returns `null` (or a specified default value).

Use this when working with semi-structured or inconsistent data where records may have varying schemas across rows.

## Examples

### Example 1: Retrieve an existing field

```powerquery
Record.FieldOrDefault([Name = "Alice", Age = 30], "Name")
```

<!--output
{"columns":[{"name":"Result","type":"text"}],"rows":[{"Result":"Alice"}]}
-->

### Example 2: Return null when field is missing

```powerquery
Record.FieldOrDefault([Name = "Alice", Age = 30], "Email")
```

<!--output
{"columns":[{"name":"Result","type":"any"}],"rows":[{"Result":null}]}
-->

### Example 3: Return a custom default when field is missing

```powerquery
Record.FieldOrDefault([Name = "Alice"], "Email", "no-email@example.com")
```

<!--output
{"columns":[{"name":"Result","type":"text"}],"rows":[{"Result":"no-email@example.com"}]}
-->

### Example 4: Safely extract an optional field from each Sales row

Input: `Sales`

```powerquery
Table.AddColumn(
    Table.SelectColumns(Table.FirstN(Sales, 3), {"OrderID"}),
    "Notes",
    each Record.FieldOrDefault(_, "Notes", ""),
    type text
)
```

<!--output
{"columns":[{"name":"OrderID","type":"number"},{"name":"Notes","type":"text"}],"rows":[{"OrderID":1,"Notes":""},{"OrderID":2,"Notes":""},{"OrderID":3,"Notes":""}]}
-->
