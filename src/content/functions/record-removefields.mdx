## Remarks

`Record.RemoveFields` returns a new record with the specified fields excluded. This is the record-level equivalent of `Table.RemoveColumns`. Use the optional `missingField` parameter to control what happens when a field you try to remove does not exist â€” by default an error is raised, but you can choose to ignore missing fields with `MissingField.Ignore`.

## Examples

### Example 1: Remove OrderDate and Region from a Sales row

Remove the OrderDate and Region fields from the first Sales row.

Input: `Sales`

```powerquery
let
    Trimmed = Record.RemoveFields(Sales{0}, {"OrderDate", "Region"})
in
    Record.ToTable(Trimmed)
```
