## Remarks

`Lines.ToBinary` is the inverse of splitting binary text content into lines. It takes a list of text lines and produces a binary value by joining the lines with the platform line separator (`#(lf)` or `#(cr,lf)`) and encoding the result.

This function is primarily used when writing text-based file content (such as CSV or log files) to binary, for example when using `Binary.Buffer` or passing to a web service.

The `encoding` parameter accepts `TextEncoding` values. The most common choice is `TextEncoding.Utf8` (the default). Set `includeByteOrderMark` to `true` when the consuming system requires a UTF-8 BOM (e.g., Excel CSV imports).

## Examples

### Example 1: Convert a list of lines to binary

```powerquery
Lines.ToBinary({"Line one", "Line two", "Line three"})
```

### Example 2: Include a UTF-8 byte order mark

```powerquery
Lines.ToBinary({"Header", "Data row"}, null, true, TextEncoding.Utf8)
```

<!--output
{"columns":[{"name":"Result","type":"binary"}],"rows":[{"Result":"77u/SGVhZGVyCkRhdGEgcm93"}]}
-->

### Example 3: Round-trip lines through binary

```powerquery
let
    original = {"apple", "banana", "cherry"},
    asBinary = Lines.ToBinary(original),
    asText = Text.FromBinary(asBinary),
    backToLines = Lines.FromText(asText)
in
    backToLines
```

<!--output
{"columns":[{"name":"Result","type":"list"}],"rows":[{"Result":["apple","banana","cherry"]}]}
-->
