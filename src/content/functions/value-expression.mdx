## Remarks

`Value.Expression` returns a structured representation (an abstract syntax tree, or AST) of the M expression that would produce the given value. The result is a record with fields describing the expression kind and its children, following the M AST schema used by the Power Query formula engine.

This function is primarily used for **metaprogramming** — building query generators, expression analyzers, or code transformation tools that operate on M expressions as data. It is used internally by the Power Query editor and by custom connector development tools.

For most data transformation tasks, `Value.Expression` is not needed. It is an advanced introspection tool.

The returned AST can be passed to `Expression.Evaluate` or `Text.FromBinary(Expression.Document(ast))` in appropriate contexts.

## Examples

### Example 1: Get the expression for a literal number

```powerquery
Value.Expression(42)
```
