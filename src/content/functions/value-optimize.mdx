## Remarks

`Value.Optimize` is an engine hint function. When called, it signals to the Power Query evaluation engine that the given value should be evaluated in an optimized fashion. The function returns the same value semantically — the output is equivalent to the input — but the engine may use different evaluation strategies such as caching, folding, or parallel execution.

In practice, `Value.Optimize` is rarely called directly by end users. It is used primarily in custom connector development and advanced scenarios where the developer wants to explicitly trigger optimization passes on a query result.

Calling `Value.Optimize` on a table or other complex value does not change the data; it only affects how the engine processes it internally.

## Examples

### Example 1: Optimize a simple number (no visible change)

```powerquery
Value.Optimize(42)
```

### Example 2: Optimize a table value

Input: `Sales`

```powerquery
let
    Filtered = Table.SelectRows(Sales, each [Quantity] > 1),
    Optimized = Value.Optimize(Filtered)
in
    Table.RowCount(Optimized)
```
