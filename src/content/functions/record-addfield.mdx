## Remarks

`Record.AddField` adds a single new field to an existing record and returns the result as a new record. The original record is not modified (records are immutable in M). If the field name already exists, an error is raised — use `Record.RemoveFields` first if you need to replace an existing field.

The optional `delayed` parameter, when set to `true`, delays evaluation of `value` until the field is first accessed. This can be useful for expensive computations you only want to run when needed.

## Examples

### Example 1: Add a new field to a record

```powerquery
Record.AddField([Name = "Alice", Age = 30], "City", "New York")
```

<!--output
{"columns":[{"name":"Name","type":"text"},{"name":"Age","type":"number"},{"name":"City","type":"text"}],"rows":[{"Name":"Alice","Age":30,"City":"New York"}]}
-->

### Example 2: Add a computed field

```powerquery
let
    R = [UnitPrice = 25, Quantity = 4]
in
    Record.AddField(R, "Total", R[UnitPrice] * R[Quantity])
```

<!--output
{"columns":[{"name":"UnitPrice","type":"number"},{"name":"Quantity","type":"number"},{"name":"Total","type":"number"}],"rows":[{"UnitPrice":25,"Quantity":4,"Total":100}]}
-->

### Example 3: Add a field to the first Sales row

Input: `Sales`

```powerquery
let
    Row = Sales{0},
    WithStatus = Record.AddField(Row, "Status", "Processed")
in
    Record.ToTable(Record.SelectFields(WithStatus, {"OrderID", "Product", "Status"}))
```

<!--output
{"columns":[{"name":"Name","type":"text"},{"name":"Value","type":"any"}],"rows":[{"Name":"OrderID","Value":1},{"Name":"Product","Value":"Widget A"},{"Name":"Status","Value":"Processed"}]}
-->
