## Remarks

`Type.Union` computes the union type of a list of types — that is, the least specific type to which all provided types are assignable. If the types are not compatible, the result is `type any`.

For example, `Type.Union({type number, type nullable number})` returns `type nullable number`, since it can hold values of either type. `Type.Union({type number, type text})` returns `type any`, as no more specific common type exists.

This function is useful in generic transformations where you need to infer a compatible type for a set of values with potentially varying types.

## Examples

### Example 1: Union of compatible numeric types

```powerquery
Type.Union({type number, type nullable number})
```
