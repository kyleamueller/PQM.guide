## Remarks

`DateTime.IsInNextSecond` returns `true` if the input `datetime` or `datetimezone` value falls within the clock second immediately following the current second (as determined by `DateTime.LocalNow()`). The current second is not included. The function is re-evaluated on each query refresh.

## Examples

### Example 1: Filter events in the next second

Keep only rows where EventTime falls in the next clock second.

```powerquery
Table.SelectRows(
    Events,
    each DateTime.IsInNextSecond([EventTime])
)
```

<!--output

### Example 2: Flag next-second events

Add a boolean column for events occurring in the next second.

```powerquery
Table.AddColumn(
    Events,
    "IsNextSecond", each DateTime.IsInNextSecond([EventTime]), type logical
)
```
