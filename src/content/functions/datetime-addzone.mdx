## Remarks

`DateTime.AddZone` annotates a `datetime` value with a UTC offset, producing a `datetimezone`. The time value itself is not adjusted — it is simply tagged with the given offset. To convert a `datetimezone` to a different offset while preserving the instant in time, use `DateTimeZone.SwitchZone`. To strip the timezone back off, use `DateTimeZone.RemoveZone`.

Use `timezoneMinutes` for non-whole-hour offsets, such as UTC+5:30 (India) or UTC+9:30 (Australia Central).

## Examples

### Example 1: Tag datetime values with Eastern Standard Time offset

Add UTC-5 to each datetime to represent EST.

Input: `Events`

```powerquery
Table.AddColumn(
    Table.SelectColumns(
        Table.FirstN(Events, 4),
        {"EventID", "EventTime"}
    ),
    "EventTimeEST",
    each DateTime.AddZone([EventTime], -5),
    type datetimezone
)
```

### Example 2: Apply a half-hour offset (UTC+5:30 for India)

Tag a datetime with the India Standard Time offset.

```powerquery
#table(
    type table [LocalTime = datetime, IST = datetimezone],
    {{#datetime(2024, 3, 15, 12, 0, 0), DateTime.AddZone(#datetime(2024, 3, 15, 12, 0, 0), 5, 30)}}
)
```

-->
