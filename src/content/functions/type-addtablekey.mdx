## Remarks

`Type.AddTableKey` adds a key definition to a table type. Keys identify columns whose values uniquely identify rows, analogous to primary and unique keys in relational databases.

In Power BI, declaring keys enables relationship management and can improve query performance. The function returns a new type — it does not modify the input type. To apply the new type to a table, use `Value.ReplaceType(table, newType)`.

A table type can have at most one primary key but multiple alternate keys.

## Examples

### Example 1: Add a primary key on a single column

```powerquery
let
    T = #table(type table [ID = number, Name = text], {{1, "Alice"}, {2, "Bob"}}),
    NewType = Type.AddTableKey(Value.Type(T), {"ID"}, true),
    Keys = Type.TableKeys(NewType)
in
    Keys{0}[Primary]
```

<!--output
{"columns":[{"name":"Result","type":"logical"}],"rows":[{"Result":true}]}
-->

### Example 2: Add a composite key

```powerquery
let
    T = #table(type table [Year = number, Month = number, Value = number], {}),
    NewType = Type.AddTableKey(Value.Type(T), {"Year", "Month"}, true),
    Keys = Type.TableKeys(NewType)
in
    Keys{0}[Columns]
```

<!--output
{"columns":[{"name":"Result","type":"list"}],"rows":[{"Result":["Year","Month"]}]}
-->
