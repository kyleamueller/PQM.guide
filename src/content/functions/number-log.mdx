## Remarks

`Number.Log` computes the logarithm of a number with respect to a specified base. When `base` is omitted, it behaves identically to `Number.Ln` (natural logarithm, base _e_). When `base` is `10`, it matches `Number.Log10`.

The number must be positive for a real result. Passing `0` returns negative infinity; a negative number returns `NaN`. The base must be positive and not equal to `1`.

## Examples

### Example 1: Base-2 logarithm (how many doublings to reach 1024)

```powerquery
Number.Log(1024, 2)
```

<!--output

### Example 2: Natural log (base e) when base is omitted

```powerquery
Number.Log(Number.E)
```

### Example 3: Base-10 log of 1000

```powerquery
Number.Log(1000, 10)
```

### Example 4: Apply log base 2 transformation to a column

Input: `Sales`

```powerquery
Table.AddColumn(
    Table.SelectColumns(Table.FirstN(Sales, 3), {"OrderID", "Revenue"}),
    "Log2Revenue", each Number.Log([Revenue], 2), type number
)
```
