## Remarks

`Value.NativeQuery` passes a native query string directly to a data source, bypassing the M query folding pipeline. This is useful when:

- You need to use database-specific SQL syntax that M cannot express or fold
- You want fine-grained control over the query sent to the data source
- Performance requires a specific query plan that M folding would not generate

The `target` is typically a database connection object, such as the result of `Sql.Database(...)`, or a table within that connection.

**Security note**: Do not construct native queries by string concatenation with user input, as this can expose your queries to SQL injection. Use parameterized queries via the `parameters` argument when incorporating external values.

By default, M cannot fold further operations on top of a native query result. Pass `[EnableFolding = true]` in `options` to allow folding (supported on connectors that implement it).

## Examples

### Example 1: Execute a SQL query against a SQL Server database

```powerquery
let
    Source = Sql.Database("myserver", "AdventureWorks"),
    Result = Value.NativeQuery(
        Source,
        "SELECT TOP 10 ProductID, Name, ListPrice FROM Production.Product ORDER BY ListPrice DESC"
    )
in
    Result
```

### Example 2: Parameterized native query

```powerquery
let
    Source = Sql.Database("myserver", "AdventureWorks"),
    MinPrice = 1000,
    Result = Value.NativeQuery(
        Source,
        "SELECT ProductID, Name, ListPrice FROM Production.Product WHERE ListPrice >= @MinPrice",
        [MinPrice = MinPrice]
    )
in
    Result
```
