## Remarks

`DateTime.FixedLocalNow` returns the current local date and time, but unlike `DateTime.LocalNow()`, it returns a consistent value for the entire duration of query evaluation. Multiple calls to `DateTime.FixedLocalNow` within a single query will always return the same timestamp, making it suitable for calculations that require a stable reference point such as age calculations or elapsed-time computations across multiple columns.

`DateTime.LocalNow()` may return different values if called at different points during a long-running query, whereas `DateTime.FixedLocalNow()` is guaranteed to be consistent.

## Examples

### Example 1: Calculate consistent elapsed time across multiple columns

Use a fixed reference to compute both days-elapsed and hours-elapsed without risk of the clock advancing between calls.

```powerquery
let
    Now = DateTime.FixedLocalNow(),
    Source = Table.SelectColumns(Table.FirstN(Events, 4), {"EventID", "EventTime"}),
    WithAge = Table.AddColumn(Source, "DaysOld",
        each Duration.TotalDays(Now - [EventTime]), type number),
    WithHours = Table.AddColumn(WithAge, "HoursOld",
        each Duration.TotalHours(Now - [EventTime]), type number)
in
    WithHours
```

### Example 2: Compare FixedLocalNow vs LocalNow behavior

Both return the current time, but FixedLocalNow is stable across calls.

```powerquery
#table(
    type table [FixedNow = datetime, Note = text],
    {{DateTime.FixedLocalNow(), "Consistent across all calls in this query"}}
)
```

<!--output
{"columns":[{"name":"FixedNow","type":"datetime"},{"name":"Note","type":"text"}],"rows":[{"FixedNow":"2026-03-08T10:00:00","Note":"Consistent across all calls in this query"}]}
