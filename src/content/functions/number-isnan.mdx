## Remarks

`Number.IsNaN` returns `true` when the input is the special IEEE 754 "Not a Number" value. NaN arises from undefined arithmetic such as `0 / 0` or taking the square root of a negative number. NaN is not equal to itself — `NaN = NaN` evaluates to `false` in M — so `Number.IsNaN` is the correct way to detect it.

Use this function to guard against NaN values before performing further calculations or writing results to a destination.

Note: `Number.IsNaN` does not accept `null` — pass a non-null number. If your column may contain nulls, filter or coalesce first.

## Examples

### Example 1: Detect NaN from zero division

```powerquery
Number.IsNaN(0 / 0)
```

### Example 2: A normal number is not NaN

```powerquery
Number.IsNaN(3.14)
```

### Example 3: Replace NaN values in a column

Input: `Sales`

```powerquery
Table.AddColumn(
    Table.SelectColumns(Table.FirstN(Sales, 4), {"OrderID", "Ratio"}),
    "SafeRatio",
    each if Number.IsNaN([Ratio]) then null else [Ratio],
    type number
)
```
