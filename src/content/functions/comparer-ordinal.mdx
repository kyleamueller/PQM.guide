## Remarks

`Comparer.Ordinal` is a built-in comparer **function** that performs case-sensitive, byte-level comparisons. It does not account for linguistic or cultural sorting rules â€” uppercase letters sort before lowercase letters (e.g., "B" comes before "a" in ordinal order).

Use `Comparer.Ordinal` as an argument to functions like `List.Sort`, `List.Distinct`, `Table.Distinct`, and `Table.Sort` when you need strict case-sensitive behavior.

## Examples

### Example 1: Case-sensitive sort of employee names

Sort employee full names using ordinal comparison. Note that uppercase names sort before lowercase names because uppercase letters have lower byte values.

Input: `Employees`

```powerquery
let
    Names = Table.Column(Employees, "FullName"),
    Sorted = List.Sort(Names, Comparer.Ordinal)
in
    #table({"FullName"}, List.Transform(Sorted, each {_}))
```
