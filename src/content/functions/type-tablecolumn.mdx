## Remarks

`Type.TableColumn` returns the declared type of a named column within a table type. This function operates on the **type** of a table, not the table itself. To obtain a table's type, use `Value.Type(table)`.

If the column does not exist in the table type, an error is raised.

This is useful for generic transformations where you need to inspect or compare column types programmatically before applying operations.

## Examples

### Example 1: Get the type of a column in a typed table

```powerquery
let
    T = #table(type table [Name = text, Age = number], {{"Alice", 30}}),
    TType = Value.Type(T),
    ColType = Type.TableColumn(TType, "Age")
in
    ColType
```

<!--output
{"columns":[{"name":"Result","type":"type"}],"rows":[{"Result":"type number"}]}

### Example 2: Check if a column type is nullable

```powerquery
let
    T = #table(type table [Score = nullable number], {{null}}),
    ColType = Type.TableColumn(Value.Type(T), "Score")
in
    Type.IsNullable(ColType)
```
