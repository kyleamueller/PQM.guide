## Remarks

`Number.Round` rounds a number to the specified number of fractional digits. When `digits` is omitted it defaults to 0 (round to the nearest integer). The optional `roundingMode` parameter controls tie-breaking behaviour â€” `RoundingMode.ToEven` (banker's rounding) is the default.

This is commonly used to clean up calculated columns such as averages, percentages, or unit conversions where excessive decimal places are not meaningful.

## Examples

### Example 1: Round salary in thousands to 1 decimal

Divide each employee's salary by 1000 and round to one decimal place.

Input: `Employees`

```powerquery
Table.AddColumn(
    Table.SelectColumns(Table.FirstN(Employees, 4), {"FullName", "Salary"}),
    "SalaryK", each Number.Round([Salary] / 1000, 1), type number
)
```

### Example 2: Round product prices to nearest ten

Round each product price to the nearest ten (digits = -1).

Input: `Products`

```powerquery
Table.AddColumn(
    Table.SelectColumns(Table.FirstN(Products, 5), {"ProductName", "Price"}),
    "RoundedPrice", each Number.Round([Price], -1), type number
)
```
