## Remarks

`Text.InferNumberType` examines a text string that represents a number and returns the most precise M type value that can hold it without loss of information. For example, `"42"` might infer `type Int8`, while `"3.14"` might infer `type Double`.

The returned value is a Power Query `type` value, not a number. It is primarily useful for dynamic type inference scenarios, such as schema detection when loading CSV data or building generic transformation functions.

The optional `culture` parameter allows locale-aware number parsing. For example, in `"de-DE"` the string `"1.234,56"` would be interpreted as 1234.56 (period as thousands separator, comma as decimal).

This is an advanced function primarily used in custom connector development and data profiling scenarios.

