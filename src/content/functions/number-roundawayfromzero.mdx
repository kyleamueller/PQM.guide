## Remarks

`Number.RoundAwayFromZero` implements the "Round Half Away From Zero" rule: when a value falls exactly halfway between two representable values, it rounds to the value farther from zero. For positive numbers this is the familiar "round half up" behavior; for negative numbers it rounds down (further negative).

This contrasts with `Number.Round`, which defaults to "Round Half to Even" (banker's rounding) to minimize statistical bias over large datasets.

Use `Number.RoundAwayFromZero` when you need conventional rounding as taught in most school curricula, or when matching Excel's `ROUND` function behavior.

## Examples

### Example 1: Round a positive tie away from zero

```powerquery
Number.RoundAwayFromZero(2.5)
```

### Example 2: Round a negative tie away from zero


### Example 3: Round to 2 decimal places


### Example 4: Round sales amounts in a table

Input: `Sales`

```powerquery
Table.AddColumn(
    Table.SelectColumns(Table.FirstN(Sales, 3), {"OrderID", "UnitPrice"}),
    "Rounded", each Number.RoundAwayFromZero([UnitPrice], 0), type number
)
```
