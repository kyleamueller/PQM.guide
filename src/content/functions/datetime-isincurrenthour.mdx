## Remarks

`DateTime.IsInCurrentHour` returns `true` if the input `datetime` or `datetimezone` value falls within the same clock hour as now (as determined by `DateTime.LocalNow()`). The hour window runs from HH:00:00 to HH:59:59.9999999. The function is re-evaluated on each query refresh.

All `DateTime.IsIn*` functions use the local system clock (`DateTime.LocalNow`) as their reference point. These fine-grained time functions are most useful in streaming or near-real-time data scenarios.

## Examples

### Example 1: Filter events from the current hour

Keep only rows where EventTime falls within the current clock hour.

```powerquery
Table.SelectRows(
    Events,
    each DateTime.IsInCurrentHour([EventTime])
)
```

### Example 2: Flag current-hour rows

Add a boolean column identifying events from the current hour.

```powerquery
Table.AddColumn(
    Events,
    "IsCurrentHour", each DateTime.IsInCurrentHour([EventTime]), type logical
)
```
