## Remarks

`Decimal.From` converts a value to the decimal number type, which uses fixed-point arithmetic to avoid the binary floating-point rounding errors that can occur with `Double.From`. The decimal type in Power Query maps to the .NET `Decimal` type with 28–29 significant digits.

This is especially important for financial and monetary values where exact decimal representation matters — for example, `0.1 + 0.2` in binary floating-point does not equal exactly `0.3`, but in decimal arithmetic it does.

Text values are parsed using the specified culture. `true` converts to `1`, `false` to `0`, `null` to `null`.

## Examples

### Example 1: Exact decimal representation

```powerquery
Decimal.From(0.1) + Decimal.From(0.2)
```

<!--output

### Example 2: Convert a text value

```powerquery
Decimal.From("1234.5678")
```

<!--output

### Example 3: Apply to a price column for exact arithmetic

Input: `Sales`

```powerquery
Table.TransformColumnTypes(
    Table.SelectColumns(Table.FirstN(Sales, 3), {"OrderID", "UnitPrice"}),
    {{"UnitPrice", Decimal.Type}}
)
```
